# 배포

***

Flowise는 플랫폼에 구애받지 않는 아키텍처로 설계되어, 다양한 배포 환경과 호환되며 사용자의 인프라 요구사항에 맞출 수 있습니다.

## 로컬 머신

Flowise를 로컬에 배포하려면 [시작하기](../../getting-started/) 가이드를 따르세요.

## 최신 클라우드 제공업체(Modern Cloud Providers)

최신 클라우드 플랫폼은 자동화를 우선시하고 개발자 워크플로우에 초점을 맞춰, 클라우드 관리와 지속적인 유지보수를 단순화합니다.&#x20;

이를 통해 필요한 기술적 전문성은 줄어들지만, 기반 인프라에 대한 커스터마이징 수준은 제한될 수 있습니다.

* [Elestio](https://elest.io/open-source/flowiseai)
* [Hugging Face](hugging-face.md)
* [Northflank](https://northflank.com/stacks/deploy-flowiseai)
* [Railway](railway.md)
* [Render](render.md)
* [Replit](replit.md)
* [RepoCloud](https://repocloud.io/details/?app\_id=29)
* [Sealos](sealos.md)
* [Zeabur](zeabur.md)

## 전통적인 클라우드 제공업체(Established Cloud Providers)

반면 전통적인 클라우드 제공업체는 특정 요구사항에 맞게 관리하고 최적화하기 위해 더 높은 수준의 기술적 전문성을 요구합니다.&#x20;

다만 이러한 복잡성은 클라우드 환경에 대한 더 큰 유연성과 제어 권한을 제공하기도 합니다.

* [AWS](aws.md)
* [Azure](azure.md)
* [DigitalOcean](digital-ocean.md)
* [GCP](gcp.md)
* [Helm을 사용한 Kubernetes](https://artifacthub.io/packages/helm/cowboysysop/flowise)

# API와 상호 작용

거의 모든 웹 애플리케이션은 RESTful API를 사용합니다. LLM이 이들과 상호 작용할 수 있도록 하면 그 실용적 유용성을 확장합니다.

이 튜토리얼은 도구 호출을 통해 LLM이 API 호출을 수행하는 방법을 보여줍니다.

## 전제 조건 - 샘플 이벤트 관리 서버

간단한 이벤트 관리 서버를 사용하고 이와 상호 작용하는 방법을 보여줍니다.

## 요청 도구

사용할 수 있는 4개의 요청 도구가 있습니다. 이를 통해 LLM은 필요할 때 GET, POST, PUT, DELETE 도구를 호출할 수 있습니다.

### 도구 1: GET (이벤트 검색)

* **목적**: API에서 이벤트 목록 또는 특정 이벤트를 가져옵니다.
* **구성 입력**: URL, 이름, 설명, 헤더, 쿼리 매개변수 스키마

### 도구 2: POST (이벤트 만들기)

* **목적**: 시스템에서 새 이벤트를 만듭니다.
* **구성 입력**: URL, 이름, 설명, 본문 스키마

### 도구 3: PUT (이벤트 업데이트)

* **목적**: 기존 이벤트의 세부 사항을 업데이트합니다.
* **구성 입력**: URL, 이름, 설명, 본문 스키마

### 도구 4: DELETE (이벤트 삭제)

* **목적**: 시스템에서 이벤트를 제거합니다.
* **구성 입력**: URL, 이름, 설명, 쿼리 매개변수 스키마

## OpenAPI 도구 키트

여러 API를 관리하기 위해 OpenAPI YAML 파일을 사용합니다. 이 도구는 YAML 파일에서 API 목록을 가져오고 각 API를 도구로 변환합니다.

## 순차적으로 API 호출

특정 작업 전후에 순차적으로 API를 호출해야 할 수 있습니다. 이 경우 HTTP 노드를 사용할 수 있습니다.

## 모범 사례

* API와 상호 작용하는 것은 에이전트가 가장 최신 정보를 가져오고 싶을 때 사용됩니다.
* 시스템 프롬프트에 명시적으로 현재 시간을 포함하는 것이 유용합니다.
* {{current_date_time}} 변수를 사용하여 현재 날짜와 시간을 검색합니다.

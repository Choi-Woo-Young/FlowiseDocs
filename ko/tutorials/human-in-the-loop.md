# 휴먼 인 더 루프

이전 튜토리얼에서는 에이전트가 도구를 동적으로 사용하여 쿼리에 답하거나 할당된 작업을 완료하는 방법을 살펴봤습니다. **휴먼 인 더 루프**는 에이전트가 계속하기 전에 인간의 입력, 승인 또는 피드백을 요청하도록 허용하여 제어 계층을 추가합니다.

휴먼 인 더 루프를 사용할 수 있는 2가지 방법이 있습니다:

* [휴먼 입력](../using-flowise/agentflowv2.md#id-11.-human-input-node) 노드를 사용하여 실행을 중단합니다.
* 에이전트의 도구에 대해 **휴먼 입력 필요** 활성화

## 휴먼 입력 노드

**휴먼 입력** 노드는 실행을 일시 중지할 수 있으며 인간이 조치를 승인하거나 거부하기 위한 피드백을 제공한 후에만 재개됩니다.

이 튜토리얼에서는 이메일을 보내기 전에 사용자 피드백을 요청하는 자동화된 이메일 응답 에이전트를 만드는 방법을 배웁니다.

### 개요

이 사용 사례의 목표는 다음을 수행할 수 있는 지능형 이메일 응답 시스템을 만드는 것입니다:

1. 들어오는 이메일 문의를 수신합니다.
2. AI를 사용하여 전문적인 이메일 응답을 생성합니다.
3. 보내기 전에 인간의 승인을 요청합니다.
4. 개선 사항 및 수정을 허용합니다.
5. 승인된 이메일을 자동으로 보냅니다.

## 에이전트 도구에서 휴먼 입력 필요

에이전트가 도구를 사용할 때 추가 체크포인트를 배치합니다. 이는 주문 배치, 예약, 회의 일정 잡기, 이메일 보내기 등 민감한 도구 호출에 중요합니다.

## 외부 검토 및 승인을 위한 실행 트레이스 공유

대시보드의 왼쪽 사이드바에서 **실행**을 클릭하면 실행 추적을 공유할 수 있습니다.

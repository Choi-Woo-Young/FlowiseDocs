# v1.3.0 마이그레이션 가이드

***

자격증명을 사용하면 모든 타사 API 키를 데이터베이스에 저장할 수 있으며 각 노드에서 쉽게 재사용할 수 있습니다. 매번 복사 붙여넣기 할 필요가 없습니다.

자격증명은 암호로 생성된 암호화 키로 암호화됩니다. 키에 대한 액세스 권한이 있는 사용자만 자격증명을 암호화/복호화할 수 있습니다. 또한 복호화된 자격증명은 네트워크 스푸핑을 피하기 위해 클라이언트로 다시 전송되지 않습니다.

다음은 v1.3.0으로 마이그레이션하는 데 도움이 되는 몇 가지 중요한 가이드입니다:

1. `PASSPHRASE` 환경 변수를 설정합니다. 이것은 자격증명을 암호화/복호화하는 데 사용되는 암호화 키를 생성하는 데 사용됩니다.
2. `SECRETKEY_PATH` 환경 변수를 설정합니다. 암호화 키를 유지하려면 암호화 키를 저장할 위치를 지정합니다.

일반적인 `.env` 파일은 다음과 같아야 합니다:

```sh
PORT=3000
PASSPHRASE=MYPASSPHRASE
DATABASE_PATH=/root/.flowise
SECRETKEY_PATH=/root/.flowise
LOG_PATH=/root/.flowise/logs
```

3. 노드 버전. 버전이 오래된 경우 노드의 오른쪽 상단에 경고 메시지가 표시됩니다. 이는 노드에 새로운 변경 사항이 있음을 의미하며 메뉴 목록에서 삭제 후 다시 추가해야 합니다.
